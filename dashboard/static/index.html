AFL++ Coverage Metrics

Edges Found — Unique control-flow edges (src→dst pairs) discovered. Primary coverage metric. More = deeper code exploration.

Bitmap Coverage % — AFL++ uses a 64KB (65,536-bucket) hash map. NOT the same as source-line coverage. 10–50% is normal and can represent excellent coverage.

Exec/s — Test cases per second. CMPLOG halves this (runs each input twice). 30–100/s with CMPLOG + complex MP4 inputs is expected.

Architecture — Local (controller, 2 instances) syncs corpus with Powerhorse (28 instances). AFL++ auto-imports sibling queue dirs, so findings are shared both ways in real time.

Source-line coverage: run make coverage inside the fuzzer container. Generates HTML report via llvm-cov.
